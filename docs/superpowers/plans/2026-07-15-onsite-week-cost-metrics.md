# Onsite Week Cost Metrics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the "Funcionários por setor" (headcount) chart on the main admin dashboard (`/admin`) with a "Custo médio de viagem de semana presencial por setor" chart, and add per-ticket cost plus a total-cost summary to the onsite-week detail page (`/admin/onsite-weeks/[id]`), with the cost summary/column animating in using the same "content that appears" motion vocabulary introduced by the `/admin/reports` continuity-transition feature.

**Architecture:** Two independent slices sharing no code. Slice 1 is a pure-data change (`src/lib/admin-analytics.ts` gains `avgOnsiteWeekCostBySector`, loses `headcountBySector`) plus its chart component and the dashboard page wiring — same pattern already used by every other sector chart on that page. Slice 2 fetches the `requests` rows tied to an `onsite_weeks.id` (already possible today via RLS policy `requests_select_own_or_admin`, no new policy needed) and threads per-request cost through to the existing `OnsiteWeekDetail` client component, which renders it with a `framer-motion` fade-in + slide-up — the same duration/delay/easing values documented in `docs/superpowers/specs/2026-07-15-admin-reports-continuity-transition-design.md`, minus the shared-element morph (out of scope here — see the design spec).

**Tech Stack:** Next.js 14 (App Router, Server Components), TypeScript strict, Supabase (Postgres + RLS, `@supabase/ssr`), Tailwind + shadcn/ui (`Table`, `Card`, `Badge`), `recharts` (via the existing `ChartContainer` wrapper) for the dashboard chart, `framer-motion` (new dependency) for the onsite-week detail animation, Vitest for pure-logic tests, npm.

## Global Constraints

- Package manager is **npm**.
- UI copy (labels, headings) is in **pt-BR**. Code identifiers (variables, types, files) are in **English**.
- This project has no React render-testing setup (no jsdom/RTL) — only pure-logic modules (`src/lib/*.ts`, no `.tsx`) get Vitest tests. Pages and components are verified via `npx tsc --noEmit`, `npm run build`, `npm run lint`, and a manual browser checklist.
- Every commit ends with the trailer `Co-Authored-By: Claude Sonnet 5 <noreply@anthropic.com>`.
- Only requests with `onsite_week_id` set **and** `status` in `["approved", "confirmed"]` count toward the sector average on the dashboard (matches the existing `spendBySector` "realized spend" convention).
- The per-ticket cost column and total-cost card on the onsite-week detail page show costs for **every** `created` outcome regardless of the request's current status — including `cancelled` (the ticket was priced/purchased at that point; cancellation doesn't erase the historical cost).
- The entrance animation reuses only the fade-in + slide-up values from `docs/superpowers/specs/2026-07-15-admin-reports-continuity-transition-design.md` (`opacity:0→1, y:10→0`, `duration:0.25, delay:0.08, ease:"easeOut"`), gated by `useReducedMotion()` from `framer-motion`. No shared-element `layoutId` morph, no change to how `/admin/onsite-weeks` navigates to `/admin/onsite-weeks/[id]` (stays a real route change via `router.push`).
- Full spec and product decisions: `docs/superpowers/specs/2026-07-15-onsite-week-cost-metrics-design.md` (do not re-litigate scope while implementing — every task below traces back to a section of that document).

export function avgOnsiteWeekCostBySector(
  requests: AdminQueueRequest[]
): { sector: Sector; average: number }[] {
  const totals = new Map<Sector, { sum: number; count: number }>(
    SECTORS.map((sector) => [sector, { sum: 0, count: 0 }])
  );
  for (const request of requests) {
    if (!request.onsite_week_id) continue;
    if (!isRealizedSpend(request)) continue;
    const key = request.corporate.cost_center as Sector;
    const entry = totals.get(key);
    if (!entry) continue;
    entry.sum += requestSpend(request);
    entry.count += 1;
  }
  return SECTORS.map((sector) => {
    const entry = totals.get(sector)!;
    return { sector, average: entry.count === 0 ? 0 : entry.sum / entry.count };
  });
}
```

import { createSupabaseServerClient } from "@/lib/supabase/server";
import { toOnsiteWeek, type OnsiteWeekRow } from "@/lib/onsite-weeks-mapper";
import { OnsiteWeekDetail, type RequestCost } from "@/components/admin/onsite-week-detail";
import { NotFoundState } from "@/components/layout/not-found-state";

export default async function OnsiteWeekDetailPage({ params }: { params: { id: string } }) {
  const supabase = createSupabaseServerClient();
  const { data: row } = await supabase.from("onsite_weeks").select("*").eq("id", params.id).single();

  if (!row) {
    return (
      <NotFoundState
        title="Semana presencial não encontrada"
        description="Ela pode ter sido removida, ou você não tem acesso a ela."
        backHref="/admin/onsite-weeks"
        backLabel="Semanas Presenciais"
      />
    );
  }

  const { data: requestRows } = await supabase
    .from("requests")
    .select("id, total_amount, total_currency")
    .eq("onsite_week_id", params.id);

  const requestCosts: Record<string, RequestCost> = {};
  for (const request of requestRows ?? []) {
    requestCosts[request.id] = { amount: request.total_amount, currency: request.total_currency };
  }

  return <OnsiteWeekDetail onsiteWeek={toOnsiteWeek(row as OnsiteWeekRow)} requestCosts={requestCosts} />;
}

      if (!response.ok) {
        toast.error(body?.error ?? "Não foi possível cancelar a semana presencial.");
        return;
      }
      toast.success("Semana presencial cancelada.");
      setCancelOpen(false);
      router.refresh();
    } catch {
      toast.error("Não foi possível cancelar a semana presencial.");
    } finally {
      setCancelling(false);
    }
  }

  return (
    <div className="flex flex-col gap-6">
      <Button variant="ghost" size="sm" asChild className="w-fit">
        <Link href="/admin/onsite-weeks">← Semanas Presenciais</Link>
      </Button>

      <div className="flex flex-wrap items-center justify-between gap-4">
        <div className="flex flex-col gap-1">
          <h1 className="text-lg font-semibold text-foreground">{SECTOR_LABELS[onsiteWeek.sector]}</h1>
          <span className="text-sm text-muted-foreground">
            {formatDate(onsiteWeek.week_start_date)} – {formatDate(onsiteWeek.week_end_date)}
          </span>
        </div>
        <div className="flex gap-2">
          <Badge variant={sectorBadge.variant}>{sectorBadge.label}</Badge>
          <Badge variant={statusBadge.variant}>{statusBadge.label}</Badge>
        </div>
      </div>

      <motion.div {...entranceMotionProps} className="w-fit">
        <Card>
          <CardHeader className="pb-2">
            <CardTitle className="text-sm font-medium text-muted-foreground">Custo total</CardTitle>
          </CardHeader>
          <CardContent>
            <span className="text-2xl font-semibold text-foreground">
              {formatCurrency(totalCost, totalCostCurrency)}
            </span>
          </CardContent>
        </Card>
      </motion.div>

      <motion.div {...entranceMotionProps}>
        <Table>
          <TableHeader>
            <TableRow>
              <TableHead>Funcionário</TableHead>
              <TableHead>Resultado</TableHead>
              <TableHead>Custo</TableHead>
            </TableRow>
          </TableHeader>
          <TableBody>
            {onsiteWeek.employee_outcomes.map((outcome) => {
              const cost = outcome.request_id ? requestCosts[outcome.request_id] : undefined;
              return (
                <TableRow key={outcome.employee_id}>
                  <TableCell className="text-foreground">{outcome.employee_name}</TableCell>
                  <TableCell>
                    {outcome.status === "created" ? (
                      <Link
                        href={`/admin/requests/${outcome.request_id}`}
                        className="text-emerald-700 hover:underline dark:text-emerald-300"
                      >
                        Solicitação aprovada
                      </Link>
                    ) : (
                      <span className="text-destructive">{outcome.error_message ?? "Falhou"}</span>
                    )}
                  </TableCell>
                  <TableCell className="text-foreground">
                    {cost ? formatCurrency(cost.amount, cost.currency) : "—"}
                  </TableCell>
                </TableRow>
              );
            })}
          </TableBody>
        </Table>
      </motion.div>

      {onsiteWeek.status !== "cancelled" ? (
        <div className="flex items-center gap-2">
          {failed.length > 0 ? (
            <Button variant="secondary" disabled={retrying} onClick={handleRetry}>
              {retrying ? "Tentando novamente..." : `Tentar novamente (${failed.length})`}
            </Button>
          ) : null}
          <Button variant="destructive" onClick={() => setCancelOpen(true)}>
            Cancelar semana presencial
          </Button>
        </div>
      ) : null}

      <Dialog open={cancelOpen} onOpenChange={setCancelOpen}>
        <DialogContent>
          <DialogHeader>
            <DialogTitle>Cancelar semana presencial</DialogTitle>
            <DialogDescription>
              Isso cancela todas as solicitações de viagem geradas por este lote. Essa ação não pode ser desfeita.
            </DialogDescription>
          </DialogHeader>
          <DialogFooter>
            <Button variant="secondary" onClick={() => setCancelOpen(false)}>
              Voltar
            </Button>
            <Button variant="destructive" disabled={cancelling} onClick={handleCancelConfirm}>
              {cancelling ? "Cancelando..." : "Cancelar semana presencial"}
            </Button>
          </DialogFooter>
        </DialogContent>
      </Dialog>
    </div>
  );
}
```

Note: `totalCost`/`totalCostCurrency` are computed from `createdCosts` (every `created` outcome that has a matching entry in `requestCosts`) regardless of the underlying request's current `status` — this is what makes cancelled onsite weeks still show their historical cost, per the Global Constraints above.
