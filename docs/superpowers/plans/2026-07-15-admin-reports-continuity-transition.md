# Admin Reports Continuity Transition Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** In `/admin/reports`, replace the current behavior — clicking an employee row inserts their report *below* the ranking table, forcing the admin to scroll — with a shared-element "continuity" transition: the employee's avatar+name flies from the table row into a fullscreen report panel that covers the content area, with a back arrow (and Esc) to return.

**Architecture:** `src/components/admin/employee-ranking-table.tsx` keeps its existing `selectedEmployeeId` local state (no routing change). A new `src/components/admin/employee-report-panel.tsx` wraps the existing `EmployeeDetail` with a header (back button + avatar/name) and is positioned `absolute inset-0` inside the ranking table's `relative` container, so it covers only the content area, not the sidebar. `framer-motion`'s `layoutId` shared-layout mechanism drives the morph: the avatar+name element carries the same `layoutId` in both the table row and the panel header, so when the panel mounts/unmounts (via `AnimatePresence`), Framer Motion animates that element's position/size between the two locations automatically. The rest of the table fades/scales via `variants`, and the panel's body content fades/slides in with a slight delay. `prefers-reduced-motion` (via `useReducedMotion()`) swaps all of this for a fast, positionless cross-fade.

**Tech Stack:** Next.js 14 App Router, React 18, TypeScript, `framer-motion` (new dependency), Tailwind, Radix UI primitives (`Avatar`, existing `Table`), `lucide-react` icons.

## Global Constraints

- This repo has no React rendering test setup (only `vitest` unit tests on `src/lib/*`, per `travel-app/docs/superpowers/specs/2026-07-13-admin-employees-directory-design.md:88-92`) — verification for every task below is `npx tsc --noEmit`, `npm run lint`, `npm test` (regression), plus a final manual browser walkthrough, not new unit tests.
- No routing changes. `selectedEmployeeId` stays local React state in `EmployeeRankingTable`; the back arrow calls a local handler, never `router.back()` or `router.push()`.
- The animation is the point of this feature, not a cosmetic afterthought (explicit user requirement). Use the exact timing/easing values from `travel-app/docs/superpowers/specs/2026-07-15-admin-reports-continuity-transition-design.md` (section "## Animação") — do not substitute defaults or "close enough" values.
- The overlay must cover only the content area (`<main>` in `travel-app/src/app/admin/layout.tsx`), never the fixed sidebar (`AppSidebar`). This is achieved structurally (the panel is `absolute inset-0` inside a `relative` wrapper that is already inside `<main>`), not by z-index tricks over the sidebar.
- `EmployeeDetail` (`travel-app/src/components/admin/employee-detail.tsx`) is not modified — its tabs already reset to `"all"` on every mount, which is what makes "don't persist the last viewed tab" work for free once the panel unmounts/remounts per selection.

```tsx
"use client";

import { useEffect, useRef } from "react";
import { ArrowLeft } from "lucide-react";
import { Avatar, AvatarFallback } from "@/components/ui/avatar";
import { Button } from "@/components/ui/button";
import { EmployeeDetail } from "@/components/admin/employee-detail";
import { initialsFromName } from "@/lib/utils";
import type { AdminQueueRequest } from "@/lib/requests-mapper";

interface EmployeeReportPanelProps {
  employeeId: string;
  employeeName: string;
  requests: AdminQueueRequest[];
  onBack: () => void;
}

export function EmployeeReportPanel({
  employeeId,
  employeeName,
  requests,
  onBack,
}: EmployeeReportPanelProps) {
  const backButtonRef = useRef<HTMLButtonElement>(null);

  useEffect(() => {
    backButtonRef.current?.focus();
  }, []);

  useEffect(() => {
    function handleKeyDown(event: KeyboardEvent) {
      if (event.key === "Escape") onBack();
    }
    window.addEventListener("keydown", handleKeyDown);
    return () => window.removeEventListener("keydown", handleKeyDown);
  }, [onBack]);

  return (
    <div className="absolute inset-0 z-10 flex flex-col overflow-auto bg-background">
      <div className="flex items-center gap-3 border-b p-4">
        <Button
          ref={backButtonRef}
          variant="ghost"
          size="icon"
          onClick={onBack}
          aria-label="Voltar para a lista"
        >
          <ArrowLeft className="size-4" />
        </Button>
        <div className="flex items-center gap-2">
          <Avatar className="h-9 w-9">
            <AvatarFallback>{initialsFromName(employeeName)}</AvatarFallback>
          </Avatar>
          <span className="font-medium text-foreground">{employeeName}</span>
        </div>
      </div>
      <div className="flex-1 p-6">
        <EmployeeDetail employeeId={employeeId} employeeName={employeeName} requests={requests} />
      </div>
    </div>
  );
}
```

- [ ] **Step 2: Replace `employee-ranking-table.tsx`**

Replace the entire contents of `travel-app/src/components/admin/employee-ranking-table.tsx` with:

```tsx
"use client";

import { useMemo, useRef, useState } from "react";
import { ArrowUpDown } from "lucide-react";
import { Avatar, AvatarFallback } from "@/components/ui/avatar";
import { Badge } from "@/components/ui/badge";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";
import { Table, TableBody, TableCell, TableHead, TableHeader, TableRow } from "@/components/ui/table";
import { EmployeeReportPanel } from "@/components/admin/employee-report-panel";
import { outOfPolicyByEmployee, spendByEmployee } from "@/lib/admin-analytics";
import { getSectorBadge, type Sector } from "@/lib/badge-variants";
import { formatCurrency } from "@/lib/offer-format";
import { initialsFromName } from "@/lib/utils";
import type { AdminQueueRequest } from "@/lib/requests-mapper";

type SortColumn = "spend" | "violations";

interface EmployeeRankingRow {
  employeeId: string;
  name: string;
  sector: Sector;
  totalSpend: number;
  violationCount: number;
}

export function EmployeeRankingTable({ requests }: { requests: AdminQueueRequest[] }) {
  const [sortColumn, setSortColumn] = useState<SortColumn>("spend");
  const [sortDir, setSortDir] = useState<"asc" | "desc">("desc");
  const [selectedEmployeeId, setSelectedEmployeeId] = useState<string | null>(null);
  const rowRefs = useRef<Map<string, HTMLTableRowElement>>(new Map());

  const sectorByEmployee = useMemo(() => {
    const map = new Map<string, Sector>();
    for (const request of requests) {
      if (!map.has(request.employee_id)) map.set(request.employee_id, request.employeeSector);
    }
    return map;
  }, [requests]);

  const rows = useMemo<EmployeeRankingRow[]>(() => {
    const spend = spendByEmployee(requests);
    const violations = outOfPolicyByEmployee(requests);
    const byEmployee = new Map<string, EmployeeRankingRow>();

    for (const entry of spend) {
      byEmployee.set(entry.employeeId, {
        employeeId: entry.employeeId,
        name: entry.name,
        sector: sectorByEmployee.get(entry.employeeId) ?? "engineering",
        totalSpend: entry.total,
        violationCount: 0,
      });
    }
    for (const entry of violations) {
      const existing = byEmployee.get(entry.employeeId);
      if (existing) {
        existing.violationCount = entry.count;
      } else {
        byEmployee.set(entry.employeeId, {
          employeeId: entry.employeeId,
          name: entry.name,
          sector: sectorByEmployee.get(entry.employeeId) ?? "engineering",
          totalSpend: 0,
          violationCount: entry.count,
        });
      }
    }
    return Array.from(byEmployee.values());
  }, [requests, sectorByEmployee]);

  const sortedRows = useMemo(() => {
    const factor = sortDir === "desc" ? -1 : 1;
    const key: keyof EmployeeRankingRow = sortColumn === "spend" ? "totalSpend" : "violationCount";
    return [...rows].sort((a, b) => (Number(a[key]) - Number(b[key])) * factor);
  }, [rows, sortColumn, sortDir]);

  function handleSort(column: SortColumn) {
    if (column === sortColumn) {
      setSortDir((dir) => (dir === "desc" ? "asc" : "desc"));
    } else {
      setSortColumn(column);
      setSortDir("desc");
    }
  }

  function handleBackToList() {
    const idToRefocus = selectedEmployeeId;
    setSelectedEmployeeId(null);
    if (idToRefocus) {
      rowRefs.current.get(idToRefocus)?.focus();
    }
  }

  const selectedEmployee = sortedRows.find((row) => row.employeeId === selectedEmployeeId);

  return (
    <div className="relative flex flex-col gap-5">
      <div>
        <Card>
          <CardHeader>
            <CardTitle>Ranking de funcionários</CardTitle>
          </CardHeader>
          <CardContent>
            <Table>
              <TableHeader>
                <TableRow>
                  <TableHead>Funcionário</TableHead>
                  <TableHead>Setor</TableHead>
                  <TableHead>
                    <button type="button" onClick={() => handleSort("spend")} className="flex items-center gap-1 font-medium">
                      Gasto total <ArrowUpDown className="size-3.5" />
                    </button>
                  </TableHead>
                  <TableHead>
                    <button type="button" onClick={() => handleSort("violations")} className="flex items-center gap-1 font-medium">
                      Desvios de política <ArrowUpDown className="size-3.5" />
                    </button>
                  </TableHead>
                </TableRow>
              </TableHeader>
              <TableBody>
                {sortedRows.map((row) => {
                  const sectorBadge = getSectorBadge(row.sector);
                  return (
                    <TableRow
                      key={row.employeeId}
                      ref={(el) => {
                        if (el) rowRefs.current.set(row.employeeId, el);
                        else rowRefs.current.delete(row.employeeId);
                      }}
                      tabIndex={-1}
                      data-state={row.employeeId === selectedEmployeeId ? "selected" : undefined}
                      onClick={() => setSelectedEmployeeId(row.employeeId)}
                      className="cursor-pointer"
                    >
                      <TableCell>
                        <div className="flex items-center gap-2">
                          <Avatar className="h-7 w-7">
                            <AvatarFallback>{initialsFromName(row.name)}</AvatarFallback>
                          </Avatar>
                          <span className="font-medium text-foreground">{row.name}</span>
                        </div>
                      </TableCell>
                      <TableCell>
                        <Badge variant={sectorBadge.variant}>{sectorBadge.label}</Badge>
                      </TableCell>
                      <TableCell>{formatCurrency(row.totalSpend, "BRL")}</TableCell>
                      <TableCell>{row.violationCount}</TableCell>
                    </TableRow>
                  );
                })}
              </TableBody>
            </Table>
          </CardContent>
        </Card>
      </div>

      {selectedEmployee ? (
        <EmployeeReportPanel
          employeeId={selectedEmployee.employeeId}
          employeeName={selectedEmployee.name}
          requests={requests}
          onBack={handleBackToList}
        />
      ) : null}
    </div>
  );
}
```

What changed vs. the original:
- Outer wrapper `<div className="flex flex-col gap-5">` → `<div className="relative flex flex-col gap-5">`, so the panel's future `absolute inset-0` (already used in `EmployeeReportPanel`) is scoped to this container, not the viewport.
- The table itself is now wrapped in a plain `<div>` (a no-op wrapper for now — Task 3 turns this into a `motion.div` with fade/scale variants; kept here so the diff in Task 3 is additive, not structural).
- `rowRefs`: a `Map` of `employeeId → <tr>` DOM node, populated via a ref callback on `TableRow`. Combined with `tabIndex={-1}` (makes the row programmatically focusable via `.focus()` without adding it to the normal Tab order — it's not meant to be a keyboard-navigable control, just a focus-restore target), this lets `handleBackToList` return focus to the row the admin originally clicked.
- `EmployeeDetail` import replaced with `EmployeeReportPanel`; the conditional render now passes `onBack={handleBackToList}` instead of rendering `EmployeeDetail` directly.

```tsx
"use client";

import { useEffect, useRef } from "react";
import { motion, useReducedMotion } from "framer-motion";
import { ArrowLeft } from "lucide-react";
import { Avatar, AvatarFallback } from "@/components/ui/avatar";
import { Button } from "@/components/ui/button";
import { EmployeeDetail } from "@/components/admin/employee-detail";
import { initialsFromName } from "@/lib/utils";
import type { AdminQueueRequest } from "@/lib/requests-mapper";

interface EmployeeReportPanelProps {
  employeeId: string;
  employeeName: string;
  requests: AdminQueueRequest[];
  onBack: () => void;
}

export function EmployeeReportPanel({
  employeeId,
  employeeName,
  requests,
  onBack,
}: EmployeeReportPanelProps) {
  const shouldReduceMotion = useReducedMotion();
  const backButtonRef = useRef<HTMLButtonElement>(null);

  useEffect(() => {
    backButtonRef.current?.focus();
  }, []);

  useEffect(() => {
    function handleKeyDown(event: KeyboardEvent) {
      if (event.key === "Escape") onBack();
    }
    window.addEventListener("keydown", handleKeyDown);
    return () => window.removeEventListener("keydown", handleKeyDown);
  }, [onBack]);

  return (
    <motion.div
      className="absolute inset-0 z-10 flex flex-col overflow-auto bg-background"
      exit={{ opacity: 0, transition: { duration: shouldReduceMotion ? 0.1 : 0.2 } }}
    >
      <div className="flex items-center gap-3 border-b p-4">
        <Button
          ref={backButtonRef}
          variant="ghost"
          size="icon"
          onClick={onBack}
          aria-label="Voltar para a lista"
        >
          <ArrowLeft className="size-4" />
        </Button>
        {shouldReduceMotion ? (
          <div className="flex items-center gap-2">
            <Avatar className="h-9 w-9">
              <AvatarFallback>{initialsFromName(employeeName)}</AvatarFallback>
            </Avatar>
            <span className="font-medium text-foreground">{employeeName}</span>
          </div>
        ) : (
          <motion.div
            layoutId={`employee-anchor-${employeeId}`}
            transition={{ type: "spring", stiffness: 300, damping: 30 }}
            className="flex items-center gap-2"
          >
            <Avatar className="h-9 w-9">
              <AvatarFallback>{initialsFromName(employeeName)}</AvatarFallback>
            </Avatar>
            <span className="font-medium text-foreground">{employeeName}</span>
          </motion.div>
        )}
      </div>
      <motion.div
        className="flex-1 p-6"
        initial={shouldReduceMotion ? false : { opacity: 0, y: 10 }}
        animate={{
          opacity: 1,
          y: 0,
          transition: {
            duration: shouldReduceMotion ? 0.1 : 0.25,
            delay: shouldReduceMotion ? 0 : 0.08,
            ease: "easeOut",
          },
        }}
        exit={{
          opacity: 0,
          y: shouldReduceMotion ? 0 : 10,
          transition: { duration: shouldReduceMotion ? 0.1 : 0.2, ease: "easeOut" },
        }}
      >
        <EmployeeDetail employeeId={employeeId} employeeName={employeeName} requests={requests} />
      </motion.div>
    </motion.div>
  );
}
```

Key points:
- `shouldReduceMotion` gates two things independently: (a) whether the avatar+name is a `motion.div` with `layoutId` at all (under reduced motion it's a plain `div` — no position/size animation is attempted, not even a fast one) and (b) the duration/delay of the opacity/slide tweens (fast cross-fade instead of the full choreography).
- The content `motion.div`'s `animate` and `exit` each carry their **own** `transition` (not a single shared `transition` prop on the component) — this is what makes opening delayed (`delay: 0.08`, content "builds after" the anchor) and closing immediate (no delay on `exit`), matching the spec's asymmetric choreography. A single shared `transition` prop would apply the same delay to both directions, which is wrong.
- The outer panel's `exit={{ opacity: 0, ... }}` has no matching `initial`/`animate` — it doesn't need an entrance animation of its own; the anchor's flight and the content's fade/slide already carry the entrance.

```tsx
                                <AvatarFallback>{initialsFromName(row.name)}</AvatarFallback>
                              </Avatar>
                              <span className="font-medium text-foreground">{row.name}</span>
                            </div>
                          ) : (
                            <motion.div layoutId={`employee-anchor-${row.employeeId}`} className="flex items-center gap-2">
                              <Avatar className="h-7 w-7">
                                <AvatarFallback>{initialsFromName(row.name)}</AvatarFallback>
                              </Avatar>
                              <span className="font-medium text-foreground">{row.name}</span>
                            </motion.div>
                          )}
                        </TableCell>
                        <TableCell>
                          <Badge variant={sectorBadge.variant}>{sectorBadge.label}</Badge>
                        </TableCell>
                        <TableCell>{formatCurrency(row.totalSpend, "BRL")}</TableCell>
                        <TableCell>{row.violationCount}</TableCell>
                      </TableRow>
                    );
                  })}
                </TableBody>
              </Table>
            </CardContent>
          </Card>
        </motion.div>

        <AnimatePresence>
          {selectedEmployee && (
            <EmployeeReportPanel
              key={selectedEmployee.employeeId}
              employeeId={selectedEmployee.employeeId}
              employeeName={selectedEmployee.name}
              requests={requests}
              onBack={handleBackToList}
            />
          )}
        </AnimatePresence>
      </div>
    </LayoutGroup>
  );
}
```

Key points:
- `LayoutGroup` scopes the `employee-anchor-*` `layoutId`s so they can't collide with any unrelated `layoutId` elsewhere on the page (there are none today — this is insulation against future additions, per the design doc's explicit decision).
- The avatar+name in each row is a `motion.div layoutId={...}` **only when motion isn't reduced** — under reduced motion it's a plain `div`, so no shared-layout position/size animation is attempted at all (not even a fast one), matching the design doc's "sem animação de posição/tamanho" requirement.
- `TABLE_VARIANTS.visible` carries `delay: 0.05` (closing: let the anchor land before the rest of the table reappears) while `TABLE_VARIANTS.hidden` has no delay (opening: rows start fading immediately) — this asymmetry is intentional, copied from the spec's choreography tables.
- `aria-hidden={selectedEmployeeId ? true : undefined}` on the table wrapper: screen readers skip the table's content while it's faded behind the panel.
- The row's `motion.div layoutId={...}` stays mounted in the table the whole time (per the design doc's "tabela permanece montada" decision) — when `EmployeeReportPanel` mounts a second element with the same `layoutId`, Framer Motion's shared-layout animation flies/crossfades between the two; when the panel unmounts (back/Esc), the row's own (still-mounted) element receives the "return flight" back to its natural position.
